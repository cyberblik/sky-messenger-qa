# Тест-план: Sky Messenger (Alpha)

## 1. Объект тестирования
Веб-мессенджер (React + TypeScript на фронте, Express + Socket.io + SQLite на бэке).
Реализовано: приветственный экран, регистрация, вход, общий чат в реальном времени, поиск пользователей.
Стадия: alpha. Документ демонстрирует QA-подход к существующему pet-проекту: тест-дизайн, ручное тестирование, оформление дефектов.

Ссылка на тестируемый проект: https://github.com/cyberblik/messenger

## 2. Окружение
- Frontend: React + Vite, порт 5173
- Backend: Node.js/Express + Socket.io, порт 5000
- БД: SQLite
- Инструменты: Postman (REST API), Chrome DevTools (Console, Network, Application)

## 3. Итоги тестирования

## 6. Дефект, который не удалось подтвердить 

**CHAT-05 / CHAT-08** — Попытка отправить сообщение от чужого имени и подключиться к чату без регистрации через отдельный HTML-клиент (минуя React-приложение).

Результат: подключение заблокировано на уровне Socket.io CORS-политики (`origin: "http://localhost:5173"` в server.js). Браузер выдаёт `Access to XMLHttpRequest... blocked by CORS policy`, так как локальный HTML-файл открывается с origin `null`.

Вывод: CORS-настройка для WebSocket-соединений в проекте реализована корректно и блокирует подключение с произвольных источников. Для полноценной проверки подмены username необходимо эмулировать клиента с того же origin (`localhost:5173`), что является следующим шагом тестирования (см. раздел "Дальнейшие шаги").
