# Sky Messenger — QA Documentation

Тестовая документация к pet-проекту **Sky Messenger** — веб-мессенджеру на React + Node.js/Express + Socket.io + SQLite.

🔗 Репозиторий с кодом проекта: https://github.com/cyberblik/messenger

## О чём этот репозиторий

Это не код самого мессенджера, а демонстрация QA-процесса вокруг него: тест-дизайн, ручное тестирование, оформление найденных дефектов. Проект находится на alpha-стадии разработки, поэтому часть найденных проблем — это известные ограничения текущей версии, а не неожиданные находки.

## Стек тестируемого проекта
- **Frontend:** React, TypeScript, Vite
- **Backend:** Node.js, Express, Socket.io
- **БД:** SQLite
- **Аутентификация:** bcrypt (хеширование паролей)

## Что внутри
- [`test-plan .md`](./test-plan.md) — тест-план, тест-кейсы по разделам (регистрация, вход, поиск, чат), результаты ручного прогона, найденные дефекты с указанием severity

## Инструменты, использованные при тестировании
- Ручное тестирование (UI)
- Postman — проверка REST API напрямую, в обход фронтенд-валидации
- Chrome DevTools (Console, Network, Application) — проверка сетевых запросов и состояния сессии

## Как запустить тестируемый проект локально
```bash
# Backend
cd backend
npm install
npm run dev      # запускается на http://localhost:5000

# Frontend (в отдельном терминале)
cd frontend
npm install
npm run dev      # запускается на http://localhost:5173
```

## Краткие итоги тестирования
- Прогнано вручную 13 тест-кейсов
- Найдено и подтверждено 7 дефектов (отсутствие лимитов длины ника/пароля, регистрозависимость username, отсутствие персистентной сессии, нереализованные личные сообщения и др.)
- Зафиксирован случай, где тест не удалось завершить из-за CORS-ограничения — с объяснением, что это показывает корректную защиту, а не баг
